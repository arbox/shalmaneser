fes_and_gfs_for_frame

given a frame, determine all FE/GF mappings, as well as FEs not mapped to GFs and GFs not mapped to FEs

GFs can only be detemined for verb targets, for other targets all FEs will be non-mapped

returns a list of hashes, each of them representing one mapping, with keys

  • fe: FE name, may be nil for GF not mapped to any FE. 

        The same FE name occurs in several mappings if 
    there were several syntactic nodes assigned for this FE

    FE name is "target" for the target.

  • gf: GF name, may be nil if this syntactic node assigned for this FE 

        does not correspond to any GF of the target

  • nodes: list of SynNode objects, all syntactic nodes assigned for 

        this FE. may be empty list

  • headnode: SynNode object, the head node for this GF or FE. 

        This is determined using the Headz class, except in the case
    when several syntactic nodes have been assigned for a target,
    in which case there's a separate heuristic
    May be nil if head couldn't be determined

  • headword: string, terminal word of the headnode 

        May be nil if head couldn't be determined

  • headindex: string, index of the headnode, counting the terminals of the sentence 

        in order of occurrence
    May be nil if head couldn't be determined

  • prep: string, preposition of the GF, may be nil

  • prepindex: string, index of preposition node, counting the terminals of the sentence 

        in order of occurrence; may be nil

  • voice: string, “active” or “passive” for verb targets, nil else

find_main_target

determine the main terminal for the target of a given frame

if there is only one syntactic node for the target, its head as determined by the Headz object is the main target.

if there are several syntactic nodes for the target, turn them into a list of terminals, then use the following heuristic: 

  if we have a verb with separate verb prefix, the verb is the main node
  otherwise, if there is a verb, the first verb is the main node
  otherwise, if there is a noun, the first noun is the main node
  otherwise, if there is an adjective, the first adjective is the main node
  otherwise the first terminal is main

if there is no syntactic node for the target, try to determine the lemma from the filename, then find a terminal matching the lemma

returns: main node(SynNode) or nil

304:

307:       # return its head terminal
308:       return get_terminal_head(targets.first)
309:

316:

321:       }
322:       # exactly one node, except for the separate verb prefix
323:       if targets.length() == 1
324:         return targets.first
325:       end
326:     
327:       # find first verb
328:       targets.each { |node|
329:         if node.part_of_speech() =~ /^V/
330:           return node
331:         end
332:       }
333:     
334:       # find first noun
335:       targets.each { |node|
336:         if node.part_of_speech() =~ /^N/
337:           return node
338:         end
339:       }
340:     
341:       # find first adjective
342:       targets.each { |node|
343:         if node.part_of_speech() =~ /^ADJ/
344:           return node
345:         end
346:       }
347:     
348:       # just return first node
349:       return targets.first()
350: 
351:     else
352:       # no syntactic node given for target:
353:       # use filename to determine lemma, 
354:       # then try to find matching terminal
355:

357:       if lemma
358:         lemma = lemma[0..3] # cut off last 2 characters to remove "en" at end of infinitive
359:         sent_obj.each_terminal { |node|

gf_fe_mapping_for_frame:

given a sentence and a frame, map all FEs of this frame to grammatical functions.

works only for verb targets.

returns: a pair [frame_props(array:string), mappings(array)] 

  frame_props contains general properties of the frame
    at the moment, there is just one feature: voice: "active" or "passive" for verb targets,
    nil for others.
  mappings: an array of tuples 
    [fe(string/nil), gf(string/nil), main_node(SynNode), nodes(array of SynNode)] where
    nodes are the syntactic nodes filling the FE/the GF, and 
    main_node is the main one among those nodes, determined according to the 
    heuristic in main_node_of_nodelist()

170:   def gf_fe_mapping_for_frame(frame_obj, # FrameNode object from SalsaTigerXML
171:                               sent_obj,  # SalsaTigerSentence object from SalsaTigerXML
172:                               filename)  # string: name of the file in which the sentence occurred, may be nil
173:                                          # Salsa file name schema is used to determine target lemma if it is missing
174:     mapping = Array.new
175: 
176:     ## 
177:     # first deal with the target
178:     main_target = find_main_target(frame_obj, sent_obj, filename)
179: 
180: 
181:     # store info on target:
182:     # - fe = "target"
183:     # - gf = nil
184:     # - main target is main_target
185:     # - all nodes: all children of frame_obj.target()

197:         # verb: also determine voice
198:         obj = TigerVerbsModule::Misc.new()
199:         if obj.is_passive(main_target)
200:           voice = "passive"
201:         else
202:           voice = "active"
203:         end
204: 
205:       when "NN", "NE"
206:         grfuncs = TigerNounsAdjectivesModule.get_all_rels_of_n(main_target, sent_obj)
207:       when "ADJA"
208:         grfuncs = TigerNounsAdjectivesModule.get_all_rels_of_a(main_target)
209:       end
210:     end
211: 
212:     # try to match with frame elements:
213:     # if an FE points to more than one syntactic node,
214:     # try to match each of them separately
215:     frame_obj.each_fe_by_name() { |fe_node|
216: 
217:       if fe_node.name.nil?
218:         next
219:       end
220: 
221:       unless fe_node.name() == "target"
222:         fe_name = fe_node.name.gsub(" ", "_")
223:

226: 
227:           # is this node listed in the list of grammatical functions?

238:             # construct a dummy tuple:

245:       end
246:     }

249:     grfuncs.each { |rel, node|

252:           rel == otherrel and node == othernode

terminal_position(node) click to toggle source

terminal_position

given a terminal, determine its position in the sentence: normally, its ID is _ so try to extract that position and return it as a string

failing that, it may be part of a splitword with ID __s so extract the number in the middle

if all that fails, return nil

node: SynNode object, a terminal of a Salsa/Tiger XML sentence

returns: a string, the position of the terminal in the sentence. 

      the string contains a number (starting at 1)
      nil if we couldn't determine the number
     # File PkgGeneral/GfFeMappingReg.rb, line 446

456:       return $1
457:     end
458:     
459:     return nil
460:   end
